向量知识库 MD 文档检索系统
========================

一、产品简介
------

本系统用于本地 / 服务器部署，支持 `.md` 格式知识库批量导入、关键词智能检索、多关键字排序、Markdown 格式渲染、富文本复制、多知识库隔离管理，适用于个人 / 团队内部 FAQ、操作手册、知识点沉淀。

内容保密、所有内容都保存在本地，备份、迁移方便。

* * *

二、环境安装（pip 方式）
==============

1. 安装 Python

------------

确保已安装 Python 3.12 及以上版本。当前使用的是：Python 3.12.10

官网：[https://www.python.org/](https://www.python.org/)

2. 安装依赖库

--------

本项目依赖 **Flask** 作为 Web 框架，使用chromedb作为向量数据库，支持mark down格式展示答案，使用以下命令安装：

```
   pip install flask chromadb markdown
```



3. 项目结构说明

---------

```
项目文件夹/
├── app.py           # 主程序
├── templates/
│   └── index.html   # 前端页面
├── knowledge/       # 知识库数据存储目录（自动生成）
└── file_md5_cache_FMW.json # 每个知识库会自动生成一个json文件
```

* * *

三、启动与运行
=======

1. 启动命令

-------

进入项目根目录，执行：

```
    python app.py
```



2. 访问系统

-------

启动成功后，浏览器打开： 



```
http://127.0.0.1/5000

http://localhost:5000
```



3. 关闭服务

-------

在终端按：CTRL + C



4. 局域网访问（可选）

------------

如需让同局域网其他电脑访问，修改 `app.py` 中运行代码：

python
    if __name__ == '__main__':
        app.run(host='0.0.0.0', port=5000, debug=True)

其他电脑通过 `本机IP:5000` 访问。

* * *

四、功能使用说明
========

1. 知识库管理

--------

* 支持创建多个独立知识库，数据互不干扰,知识库名称需要使用英文。

* 点击知识库名称切换当前库

* 上传的文件仅归属当前知识库

* 每个MD文件做一个分片，只是对问题哈希之后存入向量数据库，只对问题进行搜索，不对答案进行向量化。
2. 批量上传 MD 文件

-------------

* 支持一次性上传多个 `.md` 文档

* 文件规则：
  
  * 第一行 = 问题
  * 剩余内容 = 答案（支持完整 Markdown）

* 支持标题、加粗、列表、链接、代码块
3. 搜索与排序规则

----------

* 支持空格分隔多关键字，例如：`安装 配置 报错`

* 系统自动统计**问题中关键词命中次数**

* 次数越多，排序越靠前

* 原匹配度分值仅作参考
4. 内容展示

-------

* Markdown 自动渲染

* 链接点击**新标签页打开**

* 兼容 Typedown / Typora 单行换行

* 点击问题可展开 / 折叠答案
5. 复制功能

-------

* 点击【复制问题 + 答案】
* 复制内容为**富文本格式**
* 粘贴到 Word、WPS、飞书、钉钉可保留：加粗、段落、列表、链接
* 复制后按钮旁轻提示 “已复制”，1 秒自动消失

五、备份与迁移（重要）
===========

1. 数据存储位置

---------

所有知识库数据保存在项目目录下的：
             knowledge_base/各个知识库的文件夹

每个知识库一个独立子文件夹，文件以文本形式存储。

2. 手动备份（推荐）

-----------

 保存到其他硬盘、U 盘或云盘。

3. 迁移到新电脑

---------

1. 在新电脑安装 Python

2. 安装依赖：`  pip install flask chromadb markdown

3. 复制**完整项目文件夹**到新电脑

4. 直接启动：`python app.py`

5. 数据会自动加载，无需重新上传

6. 

----------------

如需按知识库打包导出，可直接压缩对应目录：

* 备份：压缩 `knowledge/知识库名`

* 恢复：将压缩包解压回 `knowledge/` 下即可
5. 注意事项

-------

* 迁移时**不要只复制 app.py**，必须带目录结构
* 重装系统前务必备份 `knowledge/`
* 不建议手动修改文件夹内文件，容易导致读取失败

* * *

六、MD 文件格式规范
===========

为保证系统正常解析与显示，请按以下格式编写：

md
    这是问题（必须第一行）
    这是答案内容
    **支持加粗**
    - 支持列表
    [链接文字](https://www.example.com)

七、常见问题
======

1. pip 安装很慢 / 失败

----------------

使用国内镜像加速：

bash
    pip install flask -i https://pypi.tuna.tsinghua.edu.cn/simple

2. 启动提示端口被占用

------------

修改 `app.py` 端口：

python
    app.run(port=5001)

3. 复制后格式丢失

----------

仅支持粘贴到**富文本编辑器**：

* 正常：Word、WPS、飞书、钉钉、微信

* 不正常：记事本、纯文本编辑器
4. 迁移后数据丢失

----------

检查：

* `knowledge/` 目录是否一起复制

* 目录权限是否正常

* 知识库名称是否与之前一致
5. Markdown 不换行

---------------

已内置兼容 Typedown 换行规则，使用最新版页面即可。
